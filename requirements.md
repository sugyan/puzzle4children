# 知育パズル自動生成ツール 要件

## 背景・目的
小学校低学年（**1年生・6歳前後**）の子ども向けに、「法則・パターンを見つける」タイプの知育パズルを作る。紙に印刷して鉛筆で解く前提。計算や論理推論は重くしない。

## 全体方針
- **形態**: Web ページ上で問題を生成し、ブラウザの印刷機能で A4（PDF化可）に出力する静的サイト。
- **解き方**: 印刷した紙に**鉛筆だけで書き込んで解く**。切り貼り・道具は不要。
- **解の性質**: 唯一解である必要はない。「解を出せて、その解が条件を満たしていることを子ども自身が確認できる（自己検証性）」ことを重視する。
- **生成**: すべて機械的に自動生成する。シードを与えれば同じ問題を再現できる。
- **表記**: ひらがな中心、漢字は使わない。絵記号は絵文字またはインラインSVGアイコン。

## 技術スタック / 構成
- Vite + TypeScript。描画は手書き SVG（外部チャートライブラリ不使用）。依存は最小限。
- パズルはプラグイン的に追加できる構成にする（共通インタフェース + レジストリ）。

```
puzzle4children/
├─ index.html              # エントリ。種類選択・パラメータ・印刷ボタン・出力領域
├─ package.json / tsconfig.json / vite.config.ts
└─ src/
   ├─ main.ts              # UI配線: 種類選択→生成→印刷
   ├─ types.ts             # 共通インタフェース
   ├─ rng.ts               # シード付き乱数 (mulberry32) で再現可能な生成
   ├─ svg.ts               # SVG組み立てヘルパ
   ├─ layout.ts            # A4プリント1枚分のレイアウト
   ├─ style.css            # 画面用 + @media print
   └─ puzzles/
      ├─ registry.ts       # 利用可能なジェネレータ一覧（ここに追加して拡張）
      ├─ rule-maze/{generate.ts, render.ts}   # きまりめいろ
      └─ tiling/{generate.ts, render.ts}      # しきつめ
```

### 共通インタフェース（src/types.ts）
```ts
type Difficulty = 'easy' | 'normal' | 'hard';
interface PuzzleParams { seed: number; difficulty: Difficulty; }
interface GeneratedPuzzle { svgProblem: string; svgAnswer: string; title: string; howto: string; }
interface PuzzleType {
  id: string;            // 例: 'rule-maze'
  name: string;          // 例: 'きまりめいろ'
  generate(params: PuzzleParams): GeneratedPuzzle;
}
```
`registry.ts` が `PuzzleType[]` を export。新パズルは実装してここに1行足すだけで増やせる。

## パズル1: きまりめいろ（迷路 × パターン発見）
- N×N グリッド（easy 5×5 / normal 6×6 / hard 7×7）の各マスに絵記号を置く。
- ルール = 周期パターン（easy 2種 例: 🍎→🍊 / normal 3種 🍎→🍊→🍇 …の繰り返し）。盤面上部に「この じゅんばんで すすもう」と大きく図示。スタート（左上）とゴール（右下）を明示。
- **生成**: スタート→ゴールの自己回避ウォークを引き、経路上のマス i に `cycle[i % k]` を置く（=正解パス）。残りマスは記号で埋める（分岐があってもよい＝唯一解不要）。経路長で難易度調整。
- **解き方 / 自己検証**: 記号がルール順に並びゴールに着く道を鉛筆でなぞる。順番が合っていて着けば正解、と自分で確認できる。
- **答えページ**: 正解パスを色付きでハイライト表示。

## パズル2: しきつめ（ブロック分け）
- 小さな枠（easy 4×4 / normal 4×5 / hard 5×5）を提示。下に「つかうブロック」一覧（ドミノ=2マス、L字=3マス、正方形=4マス等を色・形・個数つきで表示）。
- **解き方（鉛筆のみ）**: 枠の中を線で区切って、指定ブロックの形・個数ぴったりに分ける。切り貼りなし。
- **生成**: 枠をポリオミノで全面タイリング（ランダム＋バックトラック）し、その分割を答えとして記録。出題は「空の枠＋使うブロック一覧」。
- **自己検証**: 全マスがちょうど1回ずつ覆われ、各領域が指定の形なら正解（唯一解不要）。
- **答えページ**: 分割を色分け表示。

## UI（index.html + main.ts）
- コントロール: パズル種類セレクト / 難易度 / 枚数 / シード（ランダム or 固定）/「もんだいをつくる」「いんさつ」ボタン。
- 出力: 問題ページ群 →（最後に）答えページ群。各ページ A4。シードを隅に小さく印字（再現用）。
- 印刷: `window.print()`。`@media print` で操作UIを隠し、`page-break-after` で1枚ずつ改ページ。

## 検証手順
1. `npm install && npm run dev` で起動確認。
2. 2種を easy/normal/hard で生成し、SVGが崩れず描画されること。
3. 実際に手で解き、ルールを満たせること・答えページと整合することを確認。
4. 同一シードで同一問題が再生成されること。
5. 印刷プレビューで A4 に収まり、操作UIが消え、問題→答えの順で改ページされること。
